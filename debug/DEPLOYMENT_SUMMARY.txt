WHAT THE MIGRATION DOES
========================

1. ADD COLUMN
   - Adds last_ticket_tx_hash to canonical_users
   - Creates index for fast lookups

2. CREATE FUNCTION: tickets_tx_id_fill()
   PURPOSE: Auto-populate tx_id on tickets with REAL blockchain hashes
   
   LOGIC:
   a) Balance payments ('balance', 'balance_payment'):
      → Look up user's last_topup_tx_hash from canonical_users
      → Use that hash (users paid with their balance from a topup)
   
   b) Direct crypto payments ('coinbase', 'crypto', 'privy'):
      → Check if payment_tx_hash is already a blockchain hash
      → If not, look it up in payment_webhook_events
      → Use the real blockchain hash from THAT transaction
   
   c) Otherwise:
      → Set to NULL (NO FAKE HASHES EVER)

3. CREATE FUNCTION: update_user_topup_hash()
   PURPOSE: Track user's most recent topup transaction
   
   TRIGGERED ON: user_transactions INSERT/UPDATE
   LOGIC:
   - Only process if type = 'topup'
   - Extract blockchain hash from tx_id or webhook
   - Update canonical_users.last_topup_tx_hash

4. CREATE FUNCTION: update_user_ticket_hash()
   PURPOSE: Track user's most recent ticket purchase transaction
   
   TRIGGERED ON: tickets INSERT/UPDATE of tx_id
   LOGIC:
   - Update canonical_users.last_ticket_tx_hash with ticket's tx_id
