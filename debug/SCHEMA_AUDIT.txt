LOGIC FLOW:
===========

tickets_tx_id_fill():
1. IF tx_id is already set → skip
2. IF payment_provider = 'balance'
   → Look up user's last_topup_tx_hash
   → Use that hash (or NULL if not found)
3. ELIF payment_provider IN ('coinbase_commerce', 'coinbase', 'crypto')
   → Check if payment_tx_hash is already a blockchain hash (0x + 66 chars)
   → If yes, use it
   → If not, look it up in payment_webhook_events by charge ID
   → Use found hash (or NULL if not found)
4. ELSE
   → Set to NULL

update_user_topup_hash():
- Triggered on user_transactions INSERT/UPDATE
- Only process if type = 'topup'
- Extract blockchain hash from tx_id or webhook
- Update canonical_users.last_topup_tx_hash

update_user_ticket_hash():
- Triggered on tickets INSERT/UPDATE OF tx_id
- Update canonical_users.last_ticket_tx_hash with ticket's tx_id
